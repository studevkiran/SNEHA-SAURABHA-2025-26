## ğŸ”¢ Number Generation Logic

### How Numbers Are Created:

1. **User completes payment successfully**
2. **System generates:**
   - Confirmation ID: `SS` + database auto-increment number (padded to 4 digits)
   - Receipt Number: Just the numeric part (0001, 0002, etc.)
   - Transaction ID: From Cashfree payment gateway

3. **Example Sequence:**
   - First registration: `SS0001` (Receipt: 0001)
   - Second registration: `SS0002` (Receipt: 0002)
   - Third registration: `SS0003` (Receipt: 0003)
   - 100th registration: `SS0100` (Receipt: 0100)
   - 500th registration: `SS0500` (Receipt: 0500)

## â“ Questions & Answers

**Q: Why do we need both Registration No. and Receipt No.?**
A: Registration No. is for attendees (SS0001 is easier to remember). Receipt No. is for accounting (0001 matches standard receipt numbering).

**Q: Can we use just one number?**
A: Yes! You can use only Registration No. (SS0001) everywhere. The Receipt No. is optional and can be removed from WhatsApp message if not needed.

**Q: What if someone loses their confirmation?**
A: They can:
1. Check WhatsApp message
2. Search email
3. Contact admin with mobile number
4. Admin can look up registration by mobile/name

**Q: Is the numbering sequential?**
A: Yes, starts from SS0001 and auto-increments with each new registration.

**Q: Can we restart numbering?**
A: Yes, but not recommended. Better to continue from where you left off.
